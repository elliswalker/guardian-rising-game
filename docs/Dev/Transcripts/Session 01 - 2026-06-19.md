## How this workflow works

Ellis directs in plain English. Claude writes all code, scene files, and makes all technical decisions. Ellis never opens the Godot editor to wire nodes — all scene structure and connections are written directly as `.tscn` files. This means every decision, fix, and design choice is logged here and recoverable.

## Turn 1 — Project Design Interview

**Ellis said:** Help me design an indie game — Kingdom Series-inspired, set in Destiny universe.

**What happened:** Claude ran a structured design interview to nail down the core design pillars before writing a single line of code. Topics covered: game name, Ghost system, Guardian roster, visual style, audio, win/lose conditions, progression arc, multiplayer future-state.

**Key decisions:**
- Title: Guardian Rising
- Subgenre: Kingdom-style infinite side-scrolling defense
- Player role: Speaker / Guardian on Sparrow
- Glimmer = Kingdom coins
- Ghosts = ability system (each Ghost bonds a named Destiny Guardian NPC)
- Earth → Moon → Mars planet progression
- No tutorial — game teaches through feel
- Target: Steam + Steam Deck (controller-native); secondary: browser

**Output:** PRD v1 written to `01 - PRD.md`

**What happened:** Claude created the full project structure from scratch:

## Turn 7 — M2: Wall Building

**What was built:**
- `wall.gd` — StaticBody2D, collision_layer=2, collision_mask=0 (blocks enemies, not player)
- `build_site.gd` — Area2D with pulsing teal glow, 100 Glimmer cost, Space to build, wall replaces site on build
- Three build sites in earth_highway.tscn at x=700, 900, 1100

**Design decision from Ellis:** Kingdom allows player AND allies to move through their own walls freely. Enemies stop at walls. This requires separate collision layers.

**Collision fix:** Set wall.collision_layer=2, dreg.collision_mask=3 (ground+walls), player.collision_mask=1 (ground only). Player passes through walls; Dregs stop at them. ✅

## Turn 8 — M2: Guardian NPC — First Attempt

**What was built:**
- `guardian.gd` — CharacterBody2D with DORMANT/FOLLOWING/WAITING states
- Guardian follows player at 65px range, then breaks off to walk to encampment
- Prompt showed on proximity, Space to recruit (free)

**Ellis reported bugs:**
1. Green guardian blocks "disappear or get pushed" when player moves through
2. Recruitment should cost Glimmer — "drop them glimmer to be part of my clan"
3. Encampment Space press creates Hunter job but no guardian activates
4. "I need the guardians to be recruited still"

**Root causes identified:**
1. Guardian calling move_and_slide() in DORMANT → player (layer 1) detected by guardian (mask=1) → physics push
2. No Glimmer cost coded yet
3. Encampment Area2D collision_mask=1 can't detect guardian on collision_layer=4 → body_entered never fires → arrive_at_camp() never called
4. Guardian lazy player lookup — _ready() fires before player adds itself to the "player" group

## Turn 9 — M2: Guardian NPC — Rework

**What was changed:**
- DORMANT state: `return` before `move_and_slide()` — guardian frozen in place, zero physics interaction
- Guardian y-position in tscn changed from 548 → 600 (ground level, since physics skipped in dormant)
- Recruitment costs 75 Glimmer. Prompt: `[ SPACE ]  Recruit Guardian  —  75 ◈`
- Guardian self-calls `arrive_at_camp()` in `_travel_to_camp()` when it crosses ENCAMPMENT_X+80 — no Area2D detection needed
- Encampment body_entered for guardian removed entirely
- Lazy player lookup: `_check_recruit_prompt()` tries `get_first_node_in_group("player")` on every frame until found

**Ellis tested:** Recruitment prompt now shows ✅, Space recruits for Glimmer ✅

## Turn 10 — M2: Physics Push Fix

**Ellis reported:** After recruitment, guardian and player still push each other.

**Root cause:** Guardian in MOVING_TO_CAMP calls move_and_slide() with mask=1 → detects player on layer 1 → mutual push.

**Fix:** Move player to collision_layer=8 (bit 3). Guardian mask=1, enemy mask=3 — neither has bit 3 set, so neither detects player. Player still detects ground (mask=1 → bit 0 = ground layer).

**Area2D updates required:** glimmer_cache and build_site use body_entered to detect player → set their collision_mask=8 so they see the player on bit 3.

**Final collision layer map:**
| Bitmask | Who | Detected by |
|---------|-----|-------------|
| 1 | Ground | Player (mask=1), Guardian (mask=1), Enemy (mask=3) |
| 2 | Walls | Enemy (mask=3) |
| 4 | Guardians | nothing |
| 8 | Player | Glimmer caches (mask=8), Build sites (mask=8) |
